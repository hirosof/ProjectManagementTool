Phase 5 / Phase 6 以降のフェーズ分解が確定しました。
以下は、実装判断に用いる確定事項です。

■ Phase 5（Textual版・最小成立フェーズ）
目的：
- Textual UI として「構造を作る／再利用する」体験を成立させる
- Phase 5 単体で破綻しない最小実用ラインに限定する
- フェーズ肥大化を避ける

Phase 5 で実装する機能：

1. テンプレート機能（SubProjectテンプレート）
   - 保存 / 一覧 / 選択 / 適用の基本UI
   - 対象は SubProject のみ（Project全体は扱わない）
   - 外部依存は保存しない（検出時は警告）
   - 適用時は複製・独立（新ID採番、内部依存のみ再接続）
   - 適用時ステータスは UNSET 固定
   - dry-run によるプレビュー必須（件数サマリ＋簡易ツリー）
   - 保存形式は DB 内に固定
   - テンプレート名衝突時はエラー（手動リネーム）

2. 初回セットアップ支援
   - DB未作成/未選択時の作成・選択導線を Textual UI で提供
   - 単一DB前提（複数DB管理は行わない）

3. バックアップ導線（最小）
   - DBファイルの場所表示
   - 手動バックアップ（ファイルコピー）の手順・注意点案内
   - バックアップ機能そのものは実装しない

Phase 5 に含めないことを明示的に確定：

- 検索・絞り込み
- メモ機能（プレーンテキスト含む）
- 関連リンク
- 外部ファイル添付
- 変更履歴（ログ）
- 一括操作
- 複数DB切替
- テンプレート export/import
- テンプレート高度プレビュー（差分UI・依存グラフ等）
- 適用時ステータスの選択式（コピー / NOT_STARTED 等）

■ Phase 6 以降（拡張・運用支援フェーズ）
Phase 5 完了後に以下をまとめて扱う：

- 検索・絞り込み・フィルタ
- メモ機能（Markdown、検索連動、履歴、添付等）
- 関連リンク管理
- 外部ファイル添付（参照）
- 変更履歴（ログ）
- 一括操作
- 複数DB管理・切替
- バックアップ/復元の実装（世代管理・復元UI等）
- テンプレート export/import
- テンプレート名称衝突の自動回避
- テンプレート高度プレビュー
- 適用時ステータス初期化の選択式

補足：
- Phase 5 は「構造を作る／再利用する体験」に集中する
- 運用・記録・検索・安全性強化は Phase 6 以降に集約する
